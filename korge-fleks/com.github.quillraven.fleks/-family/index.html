A family of entities. It stores entities that have a specific configuration of components. A configuration is defined via the three IteratingSystem properties "allOf", "noneOf" and "anyOf". Each component is assigned to a unique index. That index is set in the allOf, noneOf or anyOfBitArray.

A family is an EntityListener and gets notified when an entity is added to the world or the entity's component configuration changes.

Every IteratingSystem is linked to exactly one family. Families are created by the SystemService automatically when a world gets created.

Parameters

allOf

all the components that an entity must have. Default value is null.

noneOf

all the components that an entity must not have. Default value is null.

anyOf

the components that an entity must have at least one. Default value is null.
